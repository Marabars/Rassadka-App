# Автоматическая рассадка офиса

Генерирует рассадку на новый месяц на основе файла выборов сотрудников и шаблона прошлой рассадки.

---

## Требования

- Python 3.9+
- Пакеты: `openpyxl`, `pyyaml`

Установить зависимости:

```bash
pip install openpyxl pyyaml
```

---

## Входные файлы

| Файл | Описание |
|------|----------|
| `--choices` | Excel с выборами сотрудников (из Google Таблиц). Лист с названием года, например `2026`. Строки — сотрудники, колонки — даты, значения: `офис / удаленно / отпуск / выходной`. |
| `--template` | Excel с прошлой рассадкой. Лист `График`. Служит шаблоном для визуального формата и источником исторических мест. |
| `--output` | Путь, куда сохранить итоговый файл. |
| `--office-map` | *(опционально)* Excel со списком доступных мест (один столбец, заголовок в строке 1). Если не передан — список мест берётся из шаблона. |

---

## Запуск

```bash
cd seating_app

python generate_seating.py \
  --choices "путь/к/выборам.xlsx" \
  --template "путь/к/шаблону.xlsx" \
  --output "путь/к/результату.xlsx"
```

### Пример с тестовыми файлами

```bash
python generate_seating.py \
  --choices "../Rassadka v0.1/График Май гугл Таблица Обезличенный.xlsx" \
  --template "../Rassadka v0.1/График Май Обезличенный.xlsx" \
  --output "../Rassadka v0.1/Рассадка_следующий_месяц.xlsx"
```

После запуска в консоли появится краткий отчёт:

```
Done. Assigned 900 seats across 30 days.
Issues: 0 errors, 0 warnings, 0 info
Output: ..\Rassadka v0.1\Рассадка_следующий_месяц.xlsx
```

## Что происходит внутри

1. Читает файл выборов и нормализует статусы.
2. Читает шаблон: извлекает исторические назначения (сотрудник → место).
3. Строит список предпочтительных мест для каждого сотрудника (по частоте в шаблоне).
4. Для каждого дня:
   - OFFICE-сотрудники получают места (сначала привычное, затем любое свободное).
   - Остальные статусы (`удаленно`, `отпуск`, `выходной`) — ячейка пустая.
5. Пересчитывает резерв свободных мест.
6. Копирует шаблон, записывает новые значения, добавляет лист `Validation Report`.

---

## Выходной файл

- **Основной лист** (`График`) — рассадка в том же визуальном формате, что и шаблон.
- **Лист `Validation Report`** — таблица с предупреждениями и ошибками.

Коды проблем:

| Код | Уровень | Значение |
|-----|---------|----------|
| `NO_FREE_SEAT` | ERROR | Не хватило мест на день |
| `SEAT_CONFLICT` | ERROR | Два сотрудника попали на одно место |
| `UNKNOWN_STATUS` | WARNING | Неизвестный статус в файле выборов |
| `NEW_EMPLOYEE` | INFO | Сотрудник без истории мест |

## Конфигурация (`config.yaml`)

Не нужно трогать код при изменении листов или статусов — всё настраивается здесь:

```yaml
input:
  choices_sheet_name: "2026"      # Название листа в файле выборов
  template_sheet_name: "График"   # Название листа в шаблоне

status_mapping:
  office:   ["офис", "office"]
  remote:   ["удаленно", "удалённо", "remote"]
  vacation: ["отпуск", "vacation"]
  day_off:  ["выходной", "day off"]
```

## Запуск тестов

```bash
cd seating_app
python -m pytest tests/ -v
```

Тесты покрывают: нормализацию статусов, предпочтительные места, алгоритм назначения, расчёт резерва и smoke-тест на реальных файлах.
